01

GitHubとは・GitHub Pagesとは

GitHubはファイルをクラウドで管理できるサービスです。エンジニアがプログラムのソースコードを管理するために使われますが、HTMLファイルを無料でネット公開できる「GitHub Pages」という機能が、コンテンツ配布に非常に便利です。

GitHubとGitHub Pagesの関係

サービス役割例え
GitHubファイルを保管するクラウドストレージ倉庫・Googleドライブのようなもの
GitHub PagesGitHubに置いたHTMLをURLで公開する機能倉庫の商品を店頭に並べるようなもの

サムさんの使い方

# 現在の運用構成
リポジトリ名：guides
公開URL形式：https://JUNDY5104.github.io/guides/ファイル名.html

例）
https://JUNDY5104.github.io/guides/capcut-guide.html
https://JUNDY5104.github.io/guides/zoom-guide.html
https://JUNDY5104.github.io/guides/line-official-complete.html

# ポイント
・ファイルをリポジトリにアップロードするだけでURLが自動生成される
・noindexタグを入れているので検索には引っかからない
・ファイル名を変えればURLも変わる（ランダムURLで実質非公開にできる）

02

アカウント作成の手順

※すでにアカウントをお持ちの場合はSection 03へお進みください。

  1. github.com にアクセスして Sign up（サインアップ＝新規登録）をクリックする
  2. メールアドレス・パスワード・ユーザー名を入力する。ユーザー名がURLの一部になる（例：JUNDY5104）
  3. 「Are you a student or teacher?」などのアンケートはスキップして構わない
  4. 登録メールに届いた確認コードを入力して認証する
  5. ダッシュボード（ホーム画面）が表示されれば登録完了
プランについて 登録直後はFreeプランです。GitHub Pagesの公開にはFreeプランで十分です。有料のProプランへのアップグレードは不要です。

03

リポジトリの作成

リポジトリはファイルを入れる箱（フォルダ）です。サムさんの場合は既に「guides」というリポジトリが存在しますが、新規に作る場合の手順も記録しておきます。

新規リポジトリの作成手順

  1. ログイン後、右上の + アイコン→ New repository（新規リポジトリ）をクリックする
  2. Repository name（リポジトリ名）を入力する。URLの一部になるので英小文字・ハイフンを使う（例：guides）
  3. Description（説明）は任意。空白でもOK
  4. 公開設定は ● Public（公開）を選ぶ。GitHub Pagesを使うにはPublicが必要
  5. ☑ Add a README file にチェックを入れる（リポジトリを空にしないため）
  6. Create repository（リポジトリを作成）をクリックして完了

04

ファイルのアップロード

HTMLファイルを作成したらGitHubにアップロードします。これが最も頻繁に行う操作です。

ファイルのアップロード手順

  1. github.com にログインして JUNDY5104/guides リポジトリを開く
  2. Add file（ファイルを追加）ボタンをクリックする
  3. 表示されるメニューから Upload files（ファイルをアップロード）を選択する
  4. アップロード画面が開く。HTMLファイルをドラッグ＆ドロップするか、choose your files（ファイルを選択）をクリックして選択する
  5. 画面下部の Commit changes（変更を保存）セクションが表示される。コメント欄はそのままでOK
  6. Commit changes ボタンをクリックしてアップロード完了
  7. 数秒〜1分後にURLでアクセスできるようになる
複数ファイルを一度にアップロードできます アップロード画面では複数のHTMLファイルをまとめてドラッグ＆ドロップできます。一本ずつアップロードする必要はありません。

アップロード後のURL確認

# アップロードしたファイルのURLパターン
https://JUNDY5104.github.io/guides/[ファイル名].html

# 例
ファイル名：zoom-guide.html
URL：https://JUNDY5104.github.io/guides/zoom-guide.html

ファイル名：capcut-free-checklist-8v2kx4.html
URL：https://JUNDY5104.github.io/guides/capcut-free-checklist-8v2kx4.html

# 注意：GitHub Pagesが有効になっていないとURLにアクセスできない
# 設定方法はSection 05を参照

05

GitHub Pagesの公開設定

リポジトリを作っただけではURLでアクセスできません。GitHub Pagesを有効にする設定が必要です。最初に1回だけ行えば以後は不要です。

GitHub Pages有効化の手順

  1. リポジトリのページ上部タブから Settings（設定）をクリックする
  2. 左サイドバーの Code and automation セクションにある Pages をクリックする
  3. Build and deployment（ビルドとデプロイ）セクションの Source（ソース）を Deploy from a branch（ブランチからデプロイ）に設定する
  4. その下の Branch（ブランチ）を main に変更して、フォルダは / (root) のままにする
  5. Save（保存）をクリックする
  6. ページを再読み込みすると Your site is live at https://JUNDY5104.github.io/guides/ という緑色の帯が表示される
公開まで数分かかることがあります 設定後すぐにURLにアクセスしても404エラーが出る場合があります。2〜3分待ってから再度アクセスしてください。ファイルを追加した後も反映に少し時間がかかります。

06

ファイルの更新（上書き）

ガイドの内容を修正した時や、noindexを追加した時などはファイルを更新します。同じファイル名でアップロードすれば自動的に上書きされます。

方法A：同じ名前で再アップロード（最も簡単）

  1. Section 04の手順と全く同じ操作でアップロードする
  2. 既存のファイルと同名のファイルをアップロードすると Replacing existing file（既存ファイルを置き換え）と表示される
  3. Commit changes をクリックして完了。URLは変わらない

方法B：GitHub上で直接編集する

  1. リポジトリのファイル一覧から編集したいファイル名をクリックする
  2. ファイルの内容が表示される。右上の鉛筆アイコン ✏️ Edit this file（このファイルを編集）をクリックする
  3. テキストエディタが開くので内容を直接編集する
  4. 右上の Commit changes... ボタンをクリックする
  5. 確認ダイアログが出るので再度 Commit changes をクリックして完了
方法Aを推奨します GitHub上で直接HTMLを編集するのはミスが起きやすいです。基本的にはClaude（このチャット）でファイルを修正してもらい、ダウンロードして再アップロードする方法Aが確実です。

07

ファイルの削除

有料コンテンツを誤ってアップロードした時や、不要になったファイルを削除する手順です。

  1. リポジトリのファイル一覧から削除したいファイル名をクリックする
  2. ファイルの内容が表示される。右上の ・・・（3点メニュー）をクリックする
  3. メニューから 🗑 Delete file（ファイルを削除）をクリックする
  4. 画面下部の Commit changes をクリックして削除完了
  5. 数秒後にURLにアクセスすると404エラーになることを確認する
削除は元に戻せますが注意が必要 GitHubではファイルを削除しても履歴（コミット履歴）は残ります。完全に痕跡を消すことはできません。有料コンテンツは最初からGitHubにアップロードしないよう管理することをおすすめします。

よくあるトラブル・FAQ

URLにアクセスしたら「404 File not found」が表示されます。
原因は主に3つです。①GitHub Pagesがまだ有効化されていない（Section 05の設定を確認）、②ファイルのアップロード直後でまだ反映されていない（2〜3分待つ）、③URLのファイル名が実際のファイル名と違う（大文字・小文字・拡張子を確認）。この3点を順番に確認してください。
ファイルを更新したのに古い内容が表示されます。
ブラウザのキャッシュが原因の場合がほとんどです。Ctrl+Shift+R（Mac：⌘+Shift+R）でキャッシュを無視した強制リロードを試してください。それでも解決しない場合はGitHubのリポジトリでファイルが実際に更新されているか確認してください。
間違ったファイルをアップロードしてしまいました。
Section 07の手順でファイルを削除してください。有料コンテンツを誤ってアップロードした場合は速やかに削除してください。なお、GitHubのコミット履歴には記録が残りますが、ファイルを削除すればURLではアクセスできなくなります。
Settings（設定）の中にPagesが見当たりません。
リポジトリがPrivate（非公開）になっている可能性があります。Settings → General（一般）→ Danger Zone（危険ゾーン）→「Change repository visibility」からPublicに変更してください。またはリポジトリをPublicにしていないとPagesタブが表示されない場合があります。
ファイル名を変更したいです。
GitHubではファイル名の変更も直接できます。ファイルをクリック→右上の鉛筆アイコン（Edit）をクリック→ファイル名の入力欄でファイル名を書き換える→Commit changesで保存。ただしURLも変わるので、すでに公開・共有しているURLは使えなくなります。注意してください。
複数のリポジトリを持つことはできますか？
はい、何個でも作れます。現在の「guides」リポジトリと別に、例えばCoconalaクライアント向けの非公開リポジトリを作ることもできます。ただしGitHub Pagesが使えるのはPublicリポジトリのみです（有料プランのProではPrivateでも使えます）。
